# validation-lab v2 brief

## 来源
用户直接发来的升级版决策任务，要求输出真正可执行的验证决策文档。

## 核心任务
基于上一轮已有结论，对结果进行升级重写，输出更强、更硬、更可执行的判断。

## 你的身份
你是这个项目的 validation lead / 增长策略负责人 / 首发验证负责人。
你要站在"要对试错成本和执行结果负责"的角度输出结论。

## 核心问题
对于美国市场首发，我们第一款自有品牌产品的表达，是否应该以 Version A（设计感 / 科技感 / 隐私感 / 高级生活方式表达）作为主方向？
- 如果是，应该以什么形态推进？
- 如果不是，为什么？
- 下一轮到底要验证什么？

## 6 件必须完成的事
1. 做出单义决策，不模糊
2. 明确区分三件事：已确定 / 尚未确定必须验证 / 不再验证
3. 把判断转成"可验证假设"
4. 设计"下一轮怎么测"
5. 把"美国独立站首发所需信任机制"单独列成模块
6. 语气像"内部决策备忘录"，不像咨询报告

## 必须避免的问题
- 只说"更高级""更低端"
- 只做概念对比不给执行建议
- 模糊回避选择
- 假装 A 和 B 都值得同等投入
- 只讲风格差异不讲商业后果
- 只给建议不给实验设计

## 必须重点补强的 5 点
1. 说明 Version B 的问题不只是"调性不对"，而是具体商业后果
2. 说明 Version A 的真正问题是"首屏品类识别可能不够快"
3. 真正要找的不是 A/B 二选一，而是"A 的舒适度 + 信任感 + 溢价能力 + 足够清楚的品类识别"
4. 真正影响转化的不只是表达风格，而是"表达方式 + 信任组件 + 页面正规感"
5. 下一轮问题收敛成一个具体实验

## 输出结构（必须严格按此）

1. 最终决策（一句话，单义）
2. 高层摘要（5-8 句）
3. 已确定 / 未确定 / 不再验证（三模块）
4. A/B 对比表（升级版，含所有维度）
5. 商业后果分析（A 误选 B 的后果 / 过于抽象 A 的后果）
6. Adoption blocker 拆解
7. 推荐方案
8. 美国独立站首发必须同步补齐的信任机制（单独模块）
9. 下一轮测试设计（最重要）：首屏 hero 表达测试
   - A0 = 高级生活方式强，品类提示弱
   - A1 = 高级感 + 弱品类提示
   - A2 = 高级感 + 明确品类提示
   - 控制变量：产品本体/视觉质感/价格带/信任组件/页面正规感
   - 观察指标：5-10秒理解/尴尬感/可信度/继续浏览意愿/溢价认同
   - 明确 A0赢/A1赢/A2赢 的判断标准
10. Go / No-go / Iterate（三选一）
11. 一句话结论

## 写作风格
- 像创业团队内部 memo，不像咨询顾问报告
- 清楚、硬、可执行
- 可以有判断，不要过度中立
- 每条关键判断后尽量补一句"商业上具体意味着什么"

## 背景信息（来自 round-002）
- Version A = 设计感/科技感/隐私感/高级生活方式表达
- Version B = 直接功能导向/传统成人用品表达
- 团队优势：深圳供应链快/打样改款快/内部摄影后期渲染强/高质感视觉资产/品牌感设计感
- 目标用户：美国成年消费者，重视隐私信任设计感，愿为质感付费，对表达方式敏感
- 策略边界：不 App 联动/不液体/不医疗宣称/不易复杂
- SKU 特征：可充电/非 App/非液体/非医疗/易发货/有视觉差异化/高级感设计感隐私感
